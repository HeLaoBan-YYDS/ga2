一、执行摘要

优先级最高的 5 个问题

  1. 多语言文章页 canonical 指向英文主 URL。抽查 /es/codes/grow-a-garden-2-codes 与 /th/codes/grow-a-garden-2-codes 时，页面语言分别为 es、th，但 canonical 均为 https://growagarden2wiki.wiki/codes/grow-a-garden-2-codes。这会削弱或抑制本地化页面收录。
  2. 多语言首页 hreflang 不完整。/es、/pt-BR、/th 首页只输出 hreflang="en"，缺少自引用、其他语言版本和 x-default。
  3. 多语言首页 title / description 仍为英文。西语、葡语、泰语首页均使用英文标题和英文 meta description，搜索引擎可能判断为低质量本地化或重复内容。
  4. 首页内容卡片存在明显重复。浏览器 DOM 中同一组文章标题在 “Trending Now” 区域重复出现，造成页面冗余与内部链接稀释。
  5. 前端脚本资源偏大。浏览器性能条目显示脚本传输约 2.78 MB，本地 load 约 941 ms。真实线上移动网络下可能影响 LCP/INP。

快速胜利项

  • 为所有 locale 页面生成自引用 canonical，并让 canonical 与 hreflang URL 一致。
  • 为首页补全 en、es、pt-BR、th 与 x-default alternate。
  • 将分类页 meta description 从模板句改为具体搜索意图导向文案，例如 codes、pets、crops、tools 分别写独立描述。
  • 删除首页重复文章列表或改为不同模块，例如“最新文章”和“热门攻略”使用不同数据集。

三、技术 SEO 发现

1. 多语言 canonical 配置错误 已修复 2026-06-16

影响高 → 已修复
证据/es/codes/grow-a-garden-2-codes 输出 html lang=”es”，但 canonical 为英文无 locale URL。/th/codes/grow-a-garden-2-codes 同样 canonical 到英文 URL。
修复后：/es/... → https://growagarden2wiki.wiki/es/codes/grow-a-garden-2-codes；/th/... → https://growagarden2wiki.wiki/th/codes/grow-a-garden-2-codes；/pt-BR/... → https://growagarden2wiki.wiki/pt-BR/codes/grow-a-garden-2-codes。英文页面保持无前缀。
根因src/app/[locale]/[...slug]/page.tsx 中 generateMetadata 的 pathname 变量未包含语言前缀（/${contentType}/${articleSlug} 而非 /${locale}/${contentType}/${articleSlug}），且列表页 canonical 同样缺少前缀。
修复文件src/app/[locale]/[...slug]/page.tsx（第 40–41 行列表页 canonical/OG URL；第 48–52 行详情页 canonical/OG URL）
SEO 影响Google 通常会把 canonical 视为”该页面主版本”的强信号。西语、葡语、泰语页面 canonical 到英文 URL 时，本地化版本可能不被独立索引，hreflang 集群也可能被削弱。
修复建议已完成。每个 locale 页面现在 self-canonical：/es/... canonical 到 https://growagarden2wiki.wiki/es/...，/pt-BR/... canonical 到对应葡语 URL，/th/... canonical 到对应泰语 URL。

2. 首页 hreflang 不完整，缺少 x-default

影响高
证据首页只输出 hreflang="en"。抽查 /es、/pt-BR、/th 首页也只输出英文 alternate。
SEO 影响搜索引擎难以确认语言版本之间的对应关系，可能在西语、葡语、泰语搜索结果中展示英文页面，或忽略部分 locale 页面。
修复建议所有语言首页都输出完整互链：en、es、pt-BR、th、x-default。每个页面必须包含自引用 hreflang。

3. Sitemap 有多语言 URL，但没有 hreflang 标注

影响中
证据/sitemap.xml 包含多语言 URL，但 urlset 未声明 xmlns:xhtml，每个 url 下也没有 xhtml:link alternate。
SEO 影响如果 HTML head 的 hreflang 实现不稳定，sitemap 无法提供备用信号。多语言页面规模增长后，维护成本会变高。
修复建议在 sitemap 中补充 xhtml:link rel="alternate"，或确保 HTML head 完整可靠。若 locale 页面超过数千条，推荐 sitemap 级 hreflang。

4. 前端脚本资源偏大

影响中
证据浏览器资源统计显示首页脚本传输约 2.78 MB，CSS 约 285 KB。
SEO 影响移动端真实网络下可能拖慢 LCP 和 INP，间接影响搜索体验信号与转化。
修复建议检查客户端组件边界，减少首页不必要的 JS；对文章列表、视频弹窗、语言切换等交互做懒加载；用生产构建和 Lighthouse / PageSpeed Insights 验证 Core Web Vitals。

5. 生产环境安全头可增强

影响低
证据本地响应可见 X-Powered-By: Next.js，未在本地响应中看到 HSTS 等安全头。
SEO 影响不是直接排名因素，但 HTTPS、安全与可信度会影响用户体验和平台质量。
修复建议生产环境关闭 poweredByHeader，并在 HTTPS 部署层配置 HSTS、CSP、X-Content-Type-Options 等安全头。

四、站内 SEO 发现

1. 分类页 meta description 过于模板化

影响中
证据/guide 描述为 Browse all guide guides and resources for Grow a Garden 2.，存在 “guide guides” 这类模板痕迹。/codes、/release、/community、/pets 也使用类似模板。
SEO 影响分类页通常承接高价值泛关键词，模板描述降低点击率，也无法充分覆盖搜索意图。
修复建议为每个分类页写独立描述。例如 codes 页突出 “active codes, redemption steps, rewards, update date”；pets 页突出 “rankings, abilities, best pets”。

2. 首页文章列表重复

影响中
证据DOM 标题中多篇文章出现两次，例如 “Unlock Free Rewards: The Ultimate Guide to Grow A Garden 2 Codes (June 2026)” 和多篇 guide/release/crops 文章。
SEO 影响重复模块会稀释首页主题重点，增加页面长度但不增加独特价值，同时让内部链接权重分配不够清晰。
修复建议将重复区块拆成不同意图：最新更新、热门攻略、作物指南、工具入口。每个 URL 在首页主内容中优先只出现一次。

3. 标题体系整体健康，但部分 H3 过长

影响低
证据首页有 1 个 H1，并按 H2/H3 展开；但大量文章卡片标题作为 H3，长度较长。
SEO 影响搜索引擎可理解页面结构，但过多长 H3 会让首页主题发散。
修复建议文章卡片标题可保留语义，但控制首页 H3 数量；重要模块使用 H2，卡片标题必要时用更低权重元素加结构化列表。

4. 首页 title 与 description 表现良好

影响正向
证据首页 title 43 字符，description 152 字符，包含 Wiki、Codes、Guide、Seeds、Roblox 等关键词。
建议保留当前方向。后续可测试更强点击意图标题，如加入 “Active Codes” 或 “Beginner Guide”。

六、国际化 SEO 修复模板

以文章 /codes/grow-a-garden-2-codes 为例，各语言版本应输出一致的 canonical 与 hreflang 集群：

<link rel="canonical" href="https://growagarden2wiki.wiki/es/codes/grow-a-garden-2-codes">
<link rel="alternate" hreflang="en" href="https://growagarden2wiki.wiki/codes/grow-a-garden-2-codes">
<link rel="alternate" hreflang="es" href="https://growagarden2wiki.wiki/es/codes/grow-a-garden-2-codes">
<link rel="alternate" hreflang="pt-BR" href="https://growagarden2wiki.wiki/pt-BR/codes/grow-a-garden-2-codes">
<link rel="alternate" hreflang="th" href="https://growagarden2wiki.wiki/th/codes/grow-a-garden-2-codes">
<link rel="alternate" hreflang="x-default" href="https://growagarden2wiki.wiki/codes/grow-a-garden-2-codes">
注意：canonical 必须出现在 hreflang 集群中。不要让西语、葡语、泰语页面 canonical 到英文页面，否则 locale 页面会失去独立收录资格。

八、建议验收清单

  • 用浏览器检查每个 locale 页面：document.querySelector('link[rel="canonical"]').href 是否为当前语言 URL。
  • 用浏览器检查：document.querySelectorAll('link[rel="alternate"]') 是否包含所有语言和 x-default。
  • 用 Google Rich Results Test 验证首页、分类页、文章页结构化数据。
  • 用 PageSpeed Insights 分别测试移动端与桌面端 LCP、INP、CLS。
  • 上线后在 Google Search Console 观察 “Duplicate, Google chose different canonical” 与 “Alternate page with proper canonical tag” 报告。
